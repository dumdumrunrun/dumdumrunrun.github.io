數組 Array (C)

Posted on 2021-03-06

數組是什麼？

數組又稱為陣列，用來存儲相同型態資料的資料結構；由於其以連續的方式存儲在記憶體中，有讀寫快速的特性。

什麼時候會使用到數組 Array？

在校務系統中，工程師找到上面這個程式碼片段。宣告了三個變數，用來記錄學生的學號，並將紀錄的學號於終端打印出來。恩…看起來還行，但我們還有 100 筆…不 1000 筆學生的資料還沒輸入！看來今晚要加班了…

這段例子，我們知道當有 N 個變量需要進行處理時，一個一個宣告是不切實際的，而且在迴圈（while, for, do-while）的使用上也很不方便，在這樣的情況下，選擇「數組 Array」作為存儲資料的容器，看來才是明智的。

數組的組成

讓我們用數組 Array 改寫下上方這個程式碼片段。

  • 資料型態，和變數一樣，必須要宣告我們即將要存儲的資料是何種型態。
  • 名稱（標示符 Identifier），和變數相同，必須宣告唯一的標示符，作為使用數組的名稱。
  • ‘[]’方括號，在數組宣告時，裡面的數字，表示我們要向編譯器要幾個該資料型態的空間；上方方括號中的 3，就表示要了 3 個存儲整型資料的記憶體空間。
  • ‘[]’方括號，另一個功能就是「訪問數組中的元素，索引值從 0 開始訪問第一個元素」，下方娓娓道來為什麼是「從零開始」。

為什麼數組的索引值從 0 開始？

在探討「訪問數組元素時，為什麼索引值是從 0 開始？」這件事前，我們需要知道「數組名稱代表的意義」。我們知道使用變數名稱時，可以取出其存儲的資料，這是因為「變數名稱，對應到特定一塊記憶體位置」，所以我們使用變數名稱，其實就是在使用這塊記憶體位置；同樣的事情也發生在數組身上，「數組名稱所代表的是這串數組首個元素的記憶體位置，通常就簡稱數組的首地址」。

了解數組名稱代表首地址後，那為什麼索引值是從零開始呢？簡單的解釋的話，「是因為索引值表示的是，與首地址的偏移量」，而第一個元素的記憶位置就是首地址，所以偏移量當然就為 0，而第二個元素就是，首地址再偏移 1 個宣告資料型態的長度（此處是 int ，通常為 4 Byte）…以此類推。

當講到「記憶體位置」，反應快的小夥伴，可能會聯想到「指標 pointer」。沒錯，數組名稱可以視為一種指標，不過他是「常量指標」，即我們不能更改他指向的位置！

知識點

  • 數組，以連續的方式將資料存儲在記憶體中。
  • 靜態數組初始化，若使用花括號指定元素值時，可以不用指定空間大小。
  • 數組的名稱，表示的是「記憶體位置」，也就是首個元素的位置，可視為一個「常量指標」。
  • 數組中元素，訪問時「索引值從 0 開始」。

MQL4 編程：整數資料

Posted on 2021-01-17 Edited on 2021-01-24

什麼是「程式設計」？

程式設計即是「設計解決特定問題的方法」，解決問題的方法由兩個部分組成「資料結構」和「演算法」。

資料結構 Data Structure

資料結構，就是電腦中儲存、組織資料的方式；良好的資料結構，可以提升演算法的執行效率。

就像是圖書館中，將不同領域的書籍分類、建立索引，如此一來找書時，就更加容易了。

演算法 Algorithm

演算法，就是對資料進行有限的計算步驟，最終使問題得以解決。

就像是在圖書館找書時，我們有一定的步驟：1.確認要找的書屬於「圖書十大分類法中」的哪一類。2.依照索引值順序尋找。

什麼是「內存」？

當程序要執行時，電腦會將程序加載至內存，也就是記憶體；換句話說，內存會存儲，程序運行期間所有用到的資料。

了解資料是如何存儲在內存中，才能打好程式設計的基礎。

內存中儲存資料的最基本單位為 Bit (位元)，一個 Bit 只能存放 0 或 1 兩種狀態 (即通電和沒通電兩種狀態，通電為 1 沒通電為 0)，由於 Bit 的單位太小，於是將 8 Bit 組合為一組來使用，而這樣的組合稱為 Byte (位元組)。

1 個 Bit 能表示 $2^1$ 種狀態，由此可知 8 個 Bit 也就是 1 個 Byte 能表示 $2^8=256$ 種狀態了。

什麼是「計算機編碼」？

計算機僅認得 0 和 1 兩種數字，要如何將人所習慣的語言、符號儲存、表示給電腦知道？

這時就需要「編碼」，編碼的意思就是「將資訊從一種形式轉換成另一種形式」。

以「文字編碼」舉例來說，在「美國標準資訊交換碼 (ASCII : American Standard Code for Information Interchange)」中，英文字母 A 對應到電腦中是以 $0100$ $0001_2$ (下標 2 表示二進制) 儲存。

$0100$ $0001_2$ $= 65_{10}$

編碼乍看之下就是符號的轉換，然而背後實際上的考量卻遠遠不止如此，整數的編碼經歷了以下幾個階段：

  • 原碼 sign & magnitude (淘汰)
  • 一補數 ones’ complement (淘汰)
  • 二補數 two’s complement (如今電腦)

關於編碼的知識推薦閱讀 [解讀計算機編碼｜Jserv]

無符號數 (Unsigned) 和有符號數 (Signed)

所謂的有無符號指的是「負號 -」

在計算機中「無符號數」只能用來表示「零、和正數」；而「有符號數」則能表示「負數、零、正數」。顯而易見的，同樣的資料長度，有符號數因為多了負數要表示，其所能表示數的大小就比無符號數來的小。

以 1 Byte 長度的資料來說，無符號數能表示 $0$ ~ $255$ 的整數，而以「原碼 sign & magnitude」編碼的有符號數僅能表達 $-127$ ~ $127$。

什麼是「二補數系統」？

在數學中，補數 Complement 定義如下：「兩數 (A、B) 相加等於某數 (C)，則兩數 (A、B) 互為某數 (C) 的補數」；舉例來說： $1 + 9 = 10$，則「1 的十補數為 9」或「9 的十補數為 1」。

而此處二補數所指的「2」，指的是容量 (模)，亦即單一位數所能表示的狀態數。對 1 Bit 二進位數來說，只有 0 和 1 兩種狀態 (引用”計算機編碼｜Jserv”)。

此處簡單說明如何求 2 補數，背後編碼設計思路請參考上方「推薦閱讀」。

  • 二進制中，求一數的 2 補數方法：「反轉所有位，最後加上 1」
    $0001$ $0111_2$ <- $23_{10}$
    $1110$ $1000_2$ <- 反轉位元
    $1110$ $1001_2$ <- 加上 1
    $1110$ $1001_2$ <- 得到 2 補數 $-23_{10}$

  • 加法運算方法：「求和，溢位部分捨棄」
         $0001$ $0111_2$ <- $23_{10}$
      + $1110$ $1001_2$ <- $-23_{10}$
      $1$ $0000$ $0000_2$ <- 溢位捨棄
         $0000$ $0000_2$ <- $0_{10}$

「常數 Constant」v.s.「變數 Variable」概念

常數

常數的意思是「一個數值固定不變的表示」，如「圓周率$pi$、單純的數字 10」；而在計算機中，常數指的是「程序運行過程中該符號表示的值皆固定不變」。

變數

變數的意思是「一個數值不固定的表示」，如「整數 x」(x 為整數系中任意一個數，可以是 0 可以是 -1 …)；而在計算機中，變數指的是「一個用來存儲資料的容器，依據程式設計者需求，可以儲存不同值到該符號表示中」。

「宣告 Declaration」 v.s.「定義 Definition」概念

這邊以 C 語言來說明，因在 MQL4 中 extern 被用於外部參數的表示。

宣告和定義本質為「向計算機聲明某個符號表示」，差異在於「是否有進行記憶體的分配 (給定初始值)」；由此可知定義只能一次，宣告可以多次。

  • 一般情況下，變數的宣告和定義是等價的：
  • 標頭檔 (.h) 中常見的關鍵字 extern，此時的變數宣告和定義不等價：
  • 當關鍵字 extern 宣告帶有初始化值，此時的變數宣告等同定義： 所以，回到一開始的觀念，宣告和定義的差異「是否涉及記憶體分配」。

MQL4 中的「整數型態 Integer」

MQL4 整數型態有：char, uchar, short, ushort, int, uint, long, ulong (u：unsigned 無符號數)；下方資料引用“MQL4 Data Types | MQL4 Reference”

  • char
    1 byte (8 bits) ，$2^8=256$ 種組合，可存儲 -127 ~ 128 的整數 (十進制)

  • uchar
    1 byte (8 bits)，$2^8=256$ 種組合，可存儲 0 ~ 255 的整數 (十進制)

  • short
    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 -32 768 ~ 32 767 的整數 (十進制)

  • ushort
    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 0 ~ 65 535 的整數 (十進制)

  • int
    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 -2 147 483 648 ~ 2 147 483 647 的整數 (十進制)

  • uint
    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 0 ~ 4 294 967 295 的整數 (十進制)

  • long
    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲-9 223 372 036 854 775 808 ~ 9 223 372 036 854 775 807 的整數 (十進制)

  • ulong
    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲 0 ~ 18 446 744 073 709 551 616 的整數 (十進制)

MQL4 編程：程序運行

Posted on 2021-01-01

使用 MQL4 語言進行開發時，我們需要了解 MQL4 中的程序 (Program)，以及事件 (Event) 和 事件處理器 (Event Handler) 是如何作用的。

MQL4 程序 (Program)

  • 腳本 (Script)

    「一次性」的執行編寫好的指令；用於測試代碼塊、執行簡單輔助交易功能，或其他應用。

  • 專家 (Expert Advisor)

    「循環性」的執行編寫好的指令；用於程式交易，或其他應用。

  • 指標 (Indicator)

    「循環性」的執行編寫好的指令；用於市場分析並繪製線圖 (分為主圖指標和副圖指標)，或其他應用。

預定義事件 (Event)

事件 (Event) 的意思是「某種特定的情境」，例如：滑鼠點擊圖表、鍵盤快捷鍵、程序的加載、程序的卸載…等，以上都屬於事件的一種。當事件 (Event) 發生時，就會由對應的事件處理器 (Event Handler) 來進行處理，而事件處理器本身是一個函式 (Function)，該函式中究竟要執行什麼樣的內容，就由程式設計者進行撰寫。

在 MQL4 中有 8 種預先定義好的事件 (Event)，並由對應的 8 種事件處理器 (Event Handler) 進行處理；此處的觀念：「當事件生成，則執行對應的事件處理器」。

實際上，MQL4 中還有 1 種自定義事件 (Custom Event)，此處只提及不深入，由後續的文章進行介紹。

  • Init

    「專家」或「指標」程序加載時 (Expert Advisor or Custom Indicator)，該事件生成，並由事件處理器 OnInit() 進行處理。

  • Deinit

    「專家」或「指標」程序卸載時 (Expert Advisor or Custom Indicator)，該事件生成，並由事件處理器 OnDeinit() 進行處理。

  • Start

    「腳本」程序加載後，該事件生成，並由事件處理器 OnStart() 進行處理。

  • NewTick

    「專家」程序 (Expert Advisor) 於運行期間，接收到伺服器端傳來的新報價時，該事件生成，並由事件處理器 OnTick() 進行處理。

  • Calculate

    「指標」程序 (Custom Indicator) 加載後、以及任何當前的報價更動，該事件生成，並由事件處理器 OnCalculate() 進行處理。

  • Timer

    「專家」程序 (Expert Advisor) 於運行期間，依照所設定的秒數，週期性的生成該事件，並由事件處理器 OnTimer() 進行處理。

  • Tester

    「專家」程序 (Expert Advisor) 於回測器 (Strategy Tester) 執行完畢後，該事件生成，並由事件處理器 OnTester() 進行處理。

  • ChartEvent

    圖表 (Chart) 有下列情形時，該事件生成，並由事件處理器 OnChartEvent() 進行處理。

    • 鍵盤鍵入行為
    • 圖形物件創建
    • 圖形物件刪除
    • 滑鼠點擊圖形物件
    • 滑鼠拖曳圖形物件
    • 文字編輯塊編輯結束
    • 自定義事件生成

MQL4 編程：整數資料型態

Posted on 2021-01-17

程式設計（programming），即是「給出解決特定問題的過程」；而解決問題的過程則是由「資料結構 (Data Structure)」和「演算法 (Algorithm)」組成。

以料理來說，「食譜」明確的紀錄了「食材」的處理方式和順序，最終完成一道料理；在這裡食譜和食材的關係，就類似於演算法和資料結構的關係，演算法明確的定義出資料操作的方式和順序，最終達到解決問題的結果。

要了解資料結構，首先要認識資料 (Data) 是什麼、以及基本的型態有哪些。

儲存空間單位 Bits and Bytes

電腦中，所有的資料皆是以 0 和 1 的形式保存，而 bit 和 byte 則是儲存單位的代名詞。

  • bit : 電腦中最小的儲存單位，它僅能儲存 0 或 1 兩種不同的狀態

  • byte : 當 8 個 bit 為一組時，就稱為 byte；而它能儲存 $2^8$ 種不同的狀態 (1 個 bit 能存儲 $2^1$ 種狀態，所以 8 個 bit 則能存儲 $2^8$ 種狀態)

無符號數 (Unsigned) 和有符號數 (Signed)

所謂的無符號和有符號數，在數學中意思就是「有無負號」。

以一個 8 bits (1 byte) 的整型資料來說，其二進制儲存方式為 0000 0000，共 8 位數，每一位數可以是 0 或 1，總共有 $2^8=256$ 種組合方式。

所以，如果將 8 bits 的空間拿來儲存零、和正整數，則就可以表達 0 ~ 255 的數字 (十進制)；那若是連負整數也要存入呢？

於是計算機編碼有了以下幾個時代，來解決存取、運算負整數的問題 (這邊只簡單帶過，有興趣深入的可以參考這篇深度文章 「解讀計算機編碼｜黃敬群 Jserv」)。

  1. 原碼 (sign & magnitude)

    直覺的解決方式：「將最高位元用以存儲正負號」，但很快這種方式就被淘汰了。
    這樣的表達方式，會有兩個缺點：「硬體電路設計複雜、0 的表達式有兩種 (1000 0000 和 0000 0000)。

  2. 一補數 (Ones’ complement)

    減輕了硬體電路設計複雜的問題，但 0 的表達式仍有兩種，這樣編碼方式普遍用於 1960 年代的老式電腦中。

  3. 二補數 (Two’s complement)

    硬體電路設計簡單，同時 0 存在唯一表達式。

整數資料型態 (Integer)

MQL4 整數型態有：char, uchar, short, ushort, int, uint, long, ulong (u：unsigned 無符號數)

  1. char

    1 byte (8 bits) ，$2^8=256$ 種組合，可存儲 -127 ~ 128 的整數 (十進制)

  2. uchar

    1 byte (8 bits)，$2^8=256$ 種組合，可存儲 0 ~ 255 的整數 (十進制)

  3. short

    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 -32 768 ~ 32 767 的整數 (十進制)

  4. ushort

    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 0 ~ 65 535 的整數 (十進制)

  5. int

    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 -2 147 483 648 ~ 2 147 483 647 的整數 (十進制)

  6. uint

    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 0 ~ 4 294 967 295 的整數 (十進制)

  7. long

    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲-9 223 372 036 854 775 808 ~ 9 223 372 036 854 775 807 的整數 (十進制)

  8. ulong

    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲 0 ~ 18 446 744 073 709 551 616 的整數 (十進制)

變數 (Varaible)

變數，就是「電腦記憶體儲存資料的空間，並且該空間可反覆存儲資料」；在使用一個變數前，必須先進行宣告。

宣告後，電腦會分配記憶空間給該變數，而我們就可以使用變數名來對該記憶體空間進行資料存儲。

// 雙斜線表示註解，用以解釋代碼，增加可讀性、維護性，不會對程式運行造成影響。

整型資料需注意

  1. 無號數 和 有號數 不要混用進行運算，由於其紀錄資料方式不同，會導致非預期的運算結果。

  2. 型別轉換時，注意資料丟失問題。

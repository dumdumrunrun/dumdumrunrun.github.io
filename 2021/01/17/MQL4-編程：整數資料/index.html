MQL4 編程：整數資料

Posted on 2021-01-17 Edited on 2021-01-23

什麼是「程式設計」？

程式設計即是「設計解決特定問題的方法」，解決問題的方法由兩個部分組成「資料結構」和「演算法」。

  • 資料結構 Data Structure
    資料結構，就是電腦中儲存、組織資料的方式；良好的資料結構，可以提升演算法的執行效率。
    ex. 就如同圖書館中，將不同領域的書籍分類、建立索引，如此一來找書時，就更加容易了。

  • 演算法 Algorithm
    演算法，就是對資料進行有限的計算步驟，最終使問題得以解決。
    ex. 就像在圖書館找書時，我們有一定的步驟：1.確認要找的書屬於「圖書十大分類法中」的哪一類。2.依照索引值順序尋找。

什麼是「內存」？

當程序要執行時，電腦會將程序加載至內存，也就是記憶體；換句話說，內存會存儲，程序運行期間所有用到的資料。
ex. 了解資料是如何存儲在內存中，則是我們學習程式設計的第一步

在電腦中資料皆由一連串的 0 和 1 組成，在內存中則是以通不通電表示兩種狀態，這樣的狀態定義為 bit。
ex. bit 為電腦中最小「存儲單位」，可以存儲 0 或 1 兩種狀態其中一種

當 8 個 bit 為一組時，則定義為 byte；換句話說，1 個 bit 能表示 $2^1$ 種狀態，8 個 bit 就能表示 $2^8$ 種狀態了。
ex. 1 byte = 8 bit

什麼是「計算機編碼」？

當我們的硬體能夠儲存 0 和 1 兩種狀態了，我們要如何將人所習慣的語言 (數字、文字…) 存儲在電腦中？

所以這時就需要「編碼」，編碼的意思就是「將資訊從一種形式轉換成另一種形式」。

舉例「文字編碼」的一種：美國標準資訊交換碼 (ASCII : American Standard Code for Information Interchange)，英文字母 A 在電腦中以 65 的二進制方式儲存 (100 0001)。
ex. 對電腦而言，英文字母 ‘A’ 就是 “100 0001” 這串數據

編碼乍看下就是符號的轉換，但問題來了數學上的「整數」要如何編碼？要考量的問題有什麼？
ex. 人可以理解數學中的正負數、加減運算，但電腦只理解 0 和 1，不知道什麼正負、加減；因此經歷了幾個世代的努力，「二補數系統」誕生。

在說明二補數系統前，先了解下無符號數和有符號數是什麼吧。

無符號數 (Unsigned) 和有符號數 (Signed)

  • 無符號數和有符號數，指的是「有無負號’-‘」。

  • 一個 8 bits (1 byte) 長度的資料來說，二進制存儲的方式為 0000 0000，共 8 位數，每一位數可以是 0 或 1，所以總共有 $2^8=256$ 種組合方式。

  • 這樣的資料長度，拿來表示自然數 (非負整數 0, 1, 2, 3…)，則可以表示 0 ~ 255 的十進制整數，這樣的表示法為無符號數。
    0 (十進制) = 0000 0000 (二進制無符號數)
    255 (十進制) = 1111 1111 (二進制無符號數)

  • 那麼問題來了，若要將負整數也表達進 1 byte 長度的資料呢 (有符號數)？
    最直覺的想法為，將最左邊的位元空出來作為表示正負，剩下的位元表示數字，這樣的編碼方式稱為「原碼 sign & magnitude」。
    1 (十進制) = 0000 0001 (二進制有符號數)

  • 1 (十進制) = 1000 0001 (二進制有符號數)

然而這樣的編碼方式，導致硬體電路設計複雜，且 0 的表達式有兩種 +0 和 -0 (0000 0000、1000 0000)，所以很快就被淘汰了，由補數系統取代。

接著經歷過「一補數 Ones’ complement 」的時代後，「二補數 Two’s complement」為如今電腦所使用。
ex. 一補數的出現減輕了硬體電路設計的複雜度，但 0 仍然有兩種表達式

什麼是「二補數系統」？

  • 在數學中，補數 Complement 定義如下：「兩數 (A、B) 相加等於某數 (C)，則兩數 (A、B) 互為某數 (C) 的補數」
    ex. 1 + 9 = 10；則「1 的十補數為 9」或「9 的十補數為 1」

  • 關於二補數的解釋，引用“解讀計算機編碼｜Jserv” (<-推薦深度閱讀)
    「一補數」的英語寫法為 ones’ complement，中國簡體譯作「反碼」;
    「二補數」的英語寫法為 two’s complement，中國簡體譯作「補碼」。
    注意兩者英語寫法中撇號 (apostrophe) 的位置。
    ones’ complement 是對 1 求補數，這裡的 1，本質是個有限位數構成的數值系統中所能表示出的範圍，8 位元能表示的無號最大值是 11111111，所以是一系列的 1，用 “ones”。
    至於 two’s complement 是對 2 求補數，這個 2 指的是容量 (模)，亦即單一位數能表示的狀態數。對 1 位元二進位數來說，只有 0 和 1 兩種狀態。

  • 二補數系統的優點：
    硬體電路設計簡單 (只需加法電路和二補數電路即可完成有號數的加法和減法)。
    0 有唯一表達式。

  • 在二進制下，求一個數的二補數，需反轉所有位，然後加上 1；以 8 bits 資料長度舉例 (下方以下標表示進制)：
    $0001$ $0111_2$ : ($23_{10}$)
    $1110$ $1000_2$ : (反轉)
    $1110$ $1001_2$ : (加 1)
    $1110$ $1001_2$ : ($-23_{10}$)

  • 二補數下的加法運算，為求和後，溢位部分直接捨棄；以上述 23 加上 -23 舉例：
       $0001$ $0111_2$
       $1110$ $1001_2$

$1$ $0000$ $0000_2$
最後得到 $0000$ $0000_2$ 的結果

「常數 Constant」v.s.「變數 Variable」概念

  • 常數，即「一個數值固定不變的表示」，如「圓周率\pi\、單純的數字 10」
    在計算機中，常數即「程序運行過程中該符號表示的值皆固定不變」

  • 變數，即「一個數值不固定的表示」，如「整數 x」(x 為整數系中任意一個數，可以是 0 可以是 -1 …)
    在計算機中，變數為一個「容器」用來存儲資料，依據程式設計者需求，可以儲存不同值到該符號表示中

「宣告 Declaration」 v.s.「定義 Definition」概念

這邊以 C 語言來說明，因在 MQL4 中 extern 被用於外部參數的表示。

宣告和定義本質為「向計算機聲明某個符號表示」，差異在於「是否有進行記憶體的分配 (給定初始值)」；由此可知定義只能一次，宣告可以多次。

MQL4 中的「整數型態 Integer」

MQL4 整數型態有：char, uchar, short, ushort, int, uint, long, ulong (u：unsigned 無符號數)；下方資料引用“MQL4 Data Types | MQL4 Reference”

  1. char
    1 byte (8 bits) ，$2^8=256$ 種組合，可存儲 -127 ~ 128 的整數 (十進制)

  2. uchar
    1 byte (8 bits)，$2^8=256$ 種組合，可存儲 0 ~ 255 的整數 (十進制)

  3. short
    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 -32 768 ~ 32 767 的整數 (十進制)

  4. ushort
    2 bytes (16 bits)，$2^{16}=65 536$ 種組合，可存儲 0 ~ 65 535 的整數 (十進制)

  5. int
    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 -2 147 483 648 ~ 2 147 483 647 的整數 (十進制)

  6. uint
    4 bytes (32 bits)，$2^{32}= 4 294 967 296$ 種組合，可存儲 0 ~ 4 294 967 295 的整數 (十進制)

  7. long
    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲-9 223 372 036 854 775 808 ~ 9 223 372 036 854 775 807 的整數 (十進制)

  8. ulong
    8 bytes (64 bits)，$2^{64}= 18 446 744 073 709 551 616$ 種組合，可存儲 0 ~ 18 446 744 073 709 551 616 的整數 (十進制)
